This organization is staked out so the go-quake{1,2,3} family naming holds together. The Q3 port has not started; watch go-quake1 for the active engine work. Q3 will pick up the same hand-port-from-reference pattern once the earlier ports reach the playable bar — the planned upstream is ioquake3, the maintained GPL-2.0 fork of the 1999 id Software release. Q3's renderer is GPU-shader- driven, which will lean on the go-virtio Venus (Vulkan-over-virtio) stack instead of the soft renderer Q1 uses.
